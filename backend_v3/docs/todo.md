# post-MVP фаза. Идеи по дальнейшему развитию проекта.

## Семантические связи между словами
Сейчас не поддерживается:

* Синонимы.
* Антонимоы
* Связанные слова.
* Коллокаций.
* Семей слов (derive / root).
* "Ты уже встречал похожее слово, как думаешь, что оно может обозначать?"
* «Выучить группу связанных слов».
* Скорее всего потребуется делать "скрытые отдельные словари" - отдельная таблица в БД где сохраняются слова, которые пользователь еще себе не добавил.


Хотя:
В GraphQL уже есть SenseRelation и RelationType — это future hook, но:
нет SQL-реализации,
нет сценариев,
нет UX.

## Множественные карточки на одно слово + коллекции карточек

Сейчас карточку можно привязать только к самому слову. Возможно стоит расширить функционал, чтобы можно было добавлять карточку более точечно к смыслу слова. 
карточек на отдельные смыслы,
карточек на примеры,
production / reverse cards (RU → EN, EN → RU).

Важно:
не включать всё сразу,
позволить пользователю выбрать режим.

Этот функционал стоит обернгуть в "коллекции карточек" - чтобы пользователь мог собирать карточки в отдельные коллекции (по смыслу, по произношению и т.п.)
